현재 245장은 **알찬 본 과정(002–057)** 과, 그 뒤에 **방대하지만 중복·뒤섞인 부록(058–245, 188장)** 으로 구성되어 있습니다.

## 1. 진단 — 현재 구조의 4가지 문제

**① 구현 트랙의 대규모 중복.** 동일한 '증권 AI 금융정보단말'을 최소 4번 다른 방식으로 빌드합니다.
- Python/FastAPI 풀빌드 #1: **058–073**
- Python/FastAPI 풀빌드 #2: **136–191** (#1과 거의 동일 주제 반복)
- GAS/Telegram 플랫폼 빌드(Module 1~10): **094–135** — 본 과정 027–051과 내용이 겹침
- 캡스톤 MVP 실습: **078–093**
> 예: 슬라이드 **068**과 **090**은 제목·내용이 사실상 같습니다(둘 다 "GPT 분석 엔진 및 Structured Output"). 분석 엔진·ETL·텔레그램 봇 슬라이드도 트랙마다 중복됩니다.

**② '부(部)' 번호의 비순차 배열.** 예: 슬라이드 **173 = 35부**가 슬라이드 **174 = 33부**보다 먼저 나옵니다. '부록 1'과 '부록 2'도 058→074→094→136→192로 번갈아 등장해 시퀀스가 끊깁니다.

**③ 용어 번호 라벨 오류.** '핵심 용어 13'이 슬라이드 **018(딥 리서치)** 과 **025(액션 디스패처)** 에 중복 부여돼 있습니다(용어 1~16 시리즈와 충돌). 부록 077의 '용어 10선'도 본 과정 용어와 별도로 떠 있습니다.

**④ 학습 곡선의 역설.** 작동하는 결과물(성공 체험)을 만나기 전에 **개념·용어 17장(005–021)** 을 먼저 통과해야 합니다. 비개발자 비중이 높은 'AI 아키텍트' 과정에서는 *흥미·자신감 형성*이 지연됩니다.

## 2. 재구성 철학 — 학습 여정(Learning Arc)

수강생을 **"개념 이해 → 빠른 성공 → 깊은 이해 → 확장·사업화"** 의 4막 서사로 이끕니다.

```
 [1일차]  WHY ──▶ 공통 언어 ──▶ 첫 MVP(성공!) ──▶ 데이터 레이어
 (8h)    OP        M1            M2(빠른 win)      M3
 [2일차]  분석·리포트 ──▶ 운영·보안 ──▶ 캡스톤·에이전틱
 (8h)    M4            M5             M6
 [수료후] 실전 심화 라이브러리 (Track A~H) — 자기주도 + 사내 레퍼런스
```

**핵심 장치 1 — 빠른 성공(Quick Win)을 앞으로.** 용어를 다 떼기 전에 모듈 2에서 *"내 텔레그램으로 답하는 금융봇"* 을 먼저 작동시켜 자신감을 심습니다. 이후 용어/이론은 *방금 만든 것을 설명하는 언어*로 자연스럽게 흡수됩니다.

**핵심 장치 2 — 5단계 루틴 유지.** 원본의 우수한 미시 구조(**개념 → 아키텍처 → 코드 → 실습 → 확장**)를 모든 슬라이드에서 그대로 유지합니다. 재구성은 *순서와 묶음*만 바꿉니다.

**핵심 장치 3 — 비즈니스 인사이트 훅.** 각 모듈 도입부에 "이 기능이 *어떤 금융 비즈니스 가치*를 만드는가"를 1문장으로 제시(예: 리스크 점검 → B2C 구독, 종목 리포트 → 리테일 자문 자동화).

**핵심 장치 4 — 단일 캡스톤, 다중 스택.** 흩어진 4개 풀빌드를 *하나의 캡스톤(KOStockCrewAI)* 으로 수렴시키고, 구현 스택은 **트랙 A(프로코드) / 트랙 B(로우코드)** 두 갈래로 정리해 수강생이 *자기 수준에 맞는 경로*를 선택하게 합니다.

## 4. 본 과정 상세 (16시간 운영안)

### [1일차 · 8시간]

**OP. 프롤로그 (30분)** — 오리엔테이션과 동기 부여. 표지·로드맵·아키텍트의 역할·AI 네이티브 트렌드를 *"우리가 2일 뒤 무엇을 만들 것인가"* 데모 미리보기와 함께 제시.

**M1. 개념 정립 (1.5h)** — 17개 용어를 평면 나열 대신 **3개 클러스터**로 묶어 피로도↓·연결성↑:
- *통신·데이터 기초*: API · REST · JSON · Webhook · ETL (+ Layered Architecture로 종합)
- *생성형 AI 코어*: LLM · RAG · 메타프롬프트 · 딥리서치 · OCR · NotebookLM
- *개발 패러다임*: NoCode · LowCode · Vibe Coding · Serverless · Agent Mode
- 이어서 *프롬프트·AI 제어 기초*(페르소나/ReAct/인텐트 분류기/액션 디스패처/Structured Output).
> 정비 필요: 용어 번호 재부여(현재 '용어 13' 중복), 부록 077의 '용어 10선'을 본 클러스터에 흡수.

**M2. 첫 MVP — 빠른 성공 (2h)** — *과정의 심리적 분기점.* 텔레그램 봇 생성 → 웹훅 등록 → `doPost(e)` → GPT 인텐트 분류기 → 액션 디스패처까지 **직접 작동**시켜 "내가 만든 AI가 응답한다"는 성취를 1일차 오전에 확보.

**M3. 데이터 레이어 (4h)** — ECOS·DART·KOSCOM·News·Google Sheets 연동. *"데이터 확보가 곧 서비스 경쟁력"* 메시지.

### [2일차 · 8시간]

**M4. AI 리포트 엔진 (3.5h)** — 9종 리포트 엔진(시장브리핑·뉴스요약·리스크·종목·매크로·산업·기술적·기본적·포트폴리오) + Structured Output. *각 엔진 = 잠재 상품(SKU)* 관점 제시.

**M5. 운영·보안 (2h)** — 응답 전송·관리자 화이트리스트·로그·장애 대응·신규 액션 확장. *유료 서비스의 전제조건* 프레임.

**M6. 캡스톤 & 에이전틱 진화 (2.5h)** — 통합(052) → RAG(053) → CrewAI 멀티에이전트(054) → PDF 리포트(055) → 최종 체크리스트(056) → Q&A(057). 흩어진 부록 빌드들이 *여기로 수렴*함을 명확히 안내.
