# FnXpert 텔레그램 봇 - 응답 안 되는 문제 원인과 해결 (비개발자용)

> 첨부 대화에서 "리스크 점검해줘 / 시장 브리핑 / 뉴스 점검"을 입력해도 알맞은 답이 오지 않고,
> "데이터 업데이트를 실행했습니다."만 주기적으로 올라오던 증상의 원인과 해결입니다.

## 1. 무엇이 문제였나 (쉬운 설명)

대화창을 보면 "데이터 업데이트를 실행했습니다." 메시지가 약 10~11분 간격(11:51 → 12:02 → 12:13 → 12:23 → 12:34 → 12:45)으로
규칙적으로 올라옵니다. 이건 **사용자 입력에 대한 답이 아니라, 시간마다 자동 실행되는 트리거가 보내는 메시지**입니다.

반면 사용자가 직접 입력한 "시장 브리핑", "리스크 점검해줘", "뉴스 점검"에는 **답이 거의 오지 않았습니다.**
원래 코드에 다음 세 가지 구조적 약점이 있어서 입력 메시지가 "조용히 버려질" 수 있었습니다.

1. **모든 명령을 GPT(OpenAI)에만 의존** — GPT 호출이 느리거나, 요금/한도/일시 오류가 나면 분류가 안 되고,
   그 사이 사용자는 답을 못 받습니다. (기본 명령조차 외부 API에 묶여 있음)

2. **메시지를 "조용히 무시"하는 안전장치** — 가장 큰 원인.
   - `isDuplicateUpdate_`, `isDuplicateQuestion_` 함수는 잠금(Lock)을 못 잡으면 메시지를 **'중복'으로 단정하고 그냥 버립니다(무응답).**
   - 자동 트리거(데이터 업데이트)와 웹훅이 동시에 돌면 잠금 경합이 생겨, 정상 입력이 침묵 처리될 수 있습니다.

3. **무응답 상황** — 위 1·2번이 겹치면 사용자에게 아무 답도 가지 않아 "안 된다"고 느끼게 됩니다.

## 2. 어떻게 고쳤나 (`Code.gs`)

- **키워드 규칙 분류기 추가(`classifyLocally_`)**: "리스크/점검", "시장 브리핑", "뉴스", "금리/환율" 등은
  GPT 없이도 즉시 정확히 인식합니다. GPT는 규칙이 못 알아들었을 때만 **보조**로 사용합니다.
  → 인터넷/요금/장애와 무관하게 기본 명령은 항상 동작합니다.
- **메시지를 절대 조용히 버리지 않음**: 잠금 실패 시에도 메시지를 정상 처리합니다.
- **무슨 일이 있어도 1회는 응답**: 오류가 나도 안내 메시지를 보냅니다.
- **OPENAI_API_KEY 없어도 동작**: 키가 비어 있어도 기본 명령은 정상 처리됩니다.

## 3. 적용 방법 (순서대로)

1. Apps Script 편집기를 열고, 기존 코드를 `telegram-bot/Code.gs` 내용으로 **전부 교체**합니다.
2. [프로젝트 설정 > 스크립트 속성]에 `TELEGRAM_BOT_TOKEN`, `SPREADSHEET_ID`가 있는지 확인합니다.
   (`OPENAI_API_KEY`는 있으면 좋고, 없어도 됩니다.)
3. **(매우 중요) 재배포**: [배포] > [배포 관리] > 기존 배포 연필(편집) > **버전: "새 버전"** 선택 > **배포**.
   - 이렇게 해야 텔레그램 웹훅이 가리키는 **같은 URL로 새 코드가 실행**됩니다.
   - 만약 [새 배포]로 만들면 URL이 바뀌므로, 그 새 URL로 `setWebhook`을 다시 등록해야 합니다.
4. 텔레그램에서 "리스크 점검해줘", "시장 브리핑", "뉴스 점검"을 보내 답이 오는지 확인합니다.

## 4. 빠른 점검 도구 (편집기에서 실행)

- `testClassify()` : 텔레그램 전송 없이 "키워드 규칙"이 잘 분류하는지 콘솔로 확인.
- `testLogging()`  : 시트 로깅이 되는지 확인.
- `testWebhook()`  : 가짜 입력으로 전체 흐름 확인(채팅 ID를 본인 것으로 바꾸면 실제 전송됨).

## 5. 그래도 답이 안 오면 확인할 것

- 웹훅 상태: 브라우저에서 `https://api.telegram.org/bot<토큰>/getWebhookInfo` 열어
  `last_error_message`가 있는지 확인. (URL이 현재 배포와 일치하는지도 확인)
- Apps Script [실행] 로그에서 "텔레그램 전송 실패", "OpenAI API 오류" 메시지 확인.
- CommandLog 시트에 입력이 기록되는지 확인(기록이 없으면 웹훅이 안 들어오는 것 → 재배포/웹훅 재등록).
