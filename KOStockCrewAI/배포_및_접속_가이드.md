# KOStockCrewAI — 배포 · 접속 · 다운로드 가이드

비개발자도 따라 할 수 있도록 **① 원클릭 배포(접속 URL 발급) → ② 접속/사용 → ③ 다운로드** 순서로 정리했습니다.

> ⚠️ 본 서비스 결과는 정보 제공용이며 특정 종목의 매수·매도 권유가 아닙니다.

---

## 1. 원클릭 배포 (Render) — 라이브 접속 URL 만들기

이 버튼 하나로 **FastAPI(API) + Streamlit(화면) + PostgreSQL(DB)** 3종이 한 번에 배포됩니다.
설정은 저장소 루트의 `render.yaml`(Blueprint)이 자동으로 읽힙니다.

[![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy?repo=https://github.com/inan1105/HQ_Chart)

버튼이 안 보이면 아래 주소로 접속하세요.

```
https://render.com/deploy?repo=https://github.com/inan1105/HQ_Chart
```

### 따라하기

1. 위 **Deploy to Render** 버튼을 클릭합니다.
2. Render 계정으로 로그인합니다(없으면 GitHub 계정으로 무료 가입).
3. Render가 `render.yaml`을 인식해 아래 3개 리소스를 자동 구성합니다.
   - `kostock-db` (PostgreSQL, free)
   - `kostock-api` (FastAPI 웹서비스)
   - `kostock-ui` (Streamlit 웹서비스)
4. **Apply / Create Resources**를 누릅니다.
5. (선택) 환경변수 입력 — 키가 없어도 **샘플 데이터로 동작**합니다. 실데이터를 쓰려면 대시보드 Environment에 입력:
   - `OPENAI_API_KEY`, `DART_API_KEY`, `ECOS_API_KEY`, `KOSCOM_API_KEY`, `KOSCOM_BASE_URL`
6. 빌드(Docker) 완료까지 첫 배포는 5~10분 정도 걸립니다.

> 💡 free 플랜은 일정 시간 미사용 시 잠들었다가(cold start) 다음 접속 때 다시 깨어납니다. 첫 접속이 느릴 수 있어요.

## 2. 접속 · 사용하기

배포가 끝나면 Render 대시보드의 각 서비스 상단에 공개 도메인이 표시됩니다.

| 서비스 | 주소 형태 | 용도 |
|---|---|---|
| 화면(Streamlit) | `https://kostock-ui-XXXX.onrender.com` | **교육생에게 공유할 메인 URL** |
| API(FastAPI) | `https://kostock-api-XXXX.onrender.com` | 개발자/테스트용 |
| API 문서 | `https://kostock-api-XXXX.onrender.com/docs` | 클릭만으로 API 테스트 |
| 헬스체크 | `https://kostock-api-XXXX.onrender.com/health` | 정상 동작 확인 |

### 화면(UI) 사용 순서

1. **🏠 홈** — 서비스 정상 동작 상태 확인
2. **🔑 API 키 설정** — (선택) 실데이터용 키 등록 *(값은 화면에 표시되지 않음, 설정됨/미설정만 표기)*
3. **📈 리포트 조회** — 종목명 또는 6자리 코드(예: `삼성전자` 또는 `005930`) 입력 → 1페이지 리포트 + **PDF 다운로드**

### 키 없이 바로 체험 (샘플)

API 문서(`/docs`)에서 또는 주소창에 직접:

```
GET /sample/load/005930     # 더미 데이터 적재(키 불필요)
GET /report/005930          # 분석 결과 JSON
GET /report/005930/pdf      # PDF 다운로드
GET /diagnostics?live=true  # API 키 연동 상태 점검
```

## 3. 다운로드 (소스 받기)

### 방법 A — GitHub에서 ZIP 다운로드
1. https://github.com/inan1105/HQ_Chart 접속
2. 초록색 **Code ▾** 버튼 → **Download ZIP**
3. 압축 해제 후 `KOStockCrewAI/` 폴더가 프로젝트 본체입니다.

### 방법 B — git clone
```bash
git clone https://github.com/inan1105/HQ_Chart.git
cd HQ_Chart/KOStockCrewAI
```

### 방법 C — 별도 standalone zip
교육 배포용으로 `KOStockCrewAI/`만 떼어낸 독립 패키지(zip)를 채팅으로 함께 전달했습니다.
압축을 풀면 폴더 자체가 루트인 독립 프로젝트라, 새 저장소에 그대로 올려 위 버튼으로 배포할 수 있습니다.

## 5. 문제 해결 (FAQ)

- **버튼 클릭 후 저장소가 안 보여요** → Render에서 GitHub 연동(Authorize)을 먼저 해 주세요.
- **화면은 뜨는데 데이터 호출이 안 돼요** → API 서비스(`kostock-api`)가 `Live` 상태인지, UI의 `API_BASE`가 API 호스트로 연결됐는지 확인(블루프린트가 자동 연결).
- **첫 접속이 느려요** → free 플랜 cold start입니다. 30초~1분 기다리면 깨어납니다.
- **실데이터가 안 나와요** → 키 미설정 시 샘플로 동작합니다. Render Environment에 키 입력 후 재배포하세요.
- **DB 연결 오류** → `DATABASE_URL`은 블루프린트가 자동 주입하며, 코드가 `+psycopg2`로 정규화합니다. 그래도 실패하면 DB 리소스가 `Available`인지 확인하세요.
